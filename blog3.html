JS is Compiler or Interpreter Language

First, we should know the phases in the JS Engine. That will help us to understand whether JS is a Compiler or Interpreter Language.

There are 4 phases in JS Engine - CodeInput, Parsing, Compilation and Execution.

Initially JS Engine was an Interpreter language but modern browsers use both Compiler and Interpreter togetherly. As JS engine uses Interpreter and Compiler at sametime, JS is Just in time (JIT) Compilation language. We need to deep dive in the Compilation phase and Execution phase, to understand how it is both Interpreter and Compiler. Compilation Phase and Execution phases goes hand in hand.

The parsing phase provides an Abstract Syntax tree (AST) as input to the Compilation phase. The compilation phase converts the AST into byte code with help of Interpreter and sends it to the Execution phase. While Interpreter goes line by line of code to convert it into byte code at the same time the compiler tries to optimize the code as much as it can. In this way, JS use both Compiler and Interpreter.

In some modern JS Engine, there is AOT compilation. In which the compiler and Interpreter already does its work on code and its output will be executed later on.
